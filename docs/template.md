あなたはプロダクト設計者です。次の要件を前提に、Codex実装用の cc-sdd / Kiro ブリーフを作ってください。

目的:
- ClaudeCodeが「実装ブリーフ（= /kiro:spec-init に貼る文章）」を生成する
- Codexは実装のみ（再設計禁止）
- 1回のブリーフは「1PR」で完結する粒度

制約:
- まずはMVP Phase1から着手
- PRは小さく分割（1〜2日で終わる粒度）
- 各PRブリーフに必ず含める:
  1. 対象PR名（PR#）
  2. 参照ドキュメント（例: docs/product_phase2.md のどの節）
  3. 目的
  4. 範囲（In scope / Out of scope）
  5. 技術固定（変更禁止）
  6. 受け入れ条件（AC: checkbox）
  7. 禁止事項（再設計/スコープ拡張/DB変更など）
  8. 成果物（コード/テスト/PR要約）
- 不明点がある場合は「質問は最大3つまで」欄にまとめる（それ以外は仮定して進める）

出力形式:
- まず「PR分割案一覧（PR#1〜）」を箇条書き
- 次に「PR#1 の /kiro:spec-init への貼り付け文」をMarkdownで1つだけ全文出力
- 追加で「PR#2以降のブリーフは見出しだけ（本文なし）」

# 共通ルール（全PR）
- 既存の仕様・データ・UIを勝手に拡張しない（このPRのIn scopeのみ）。
- 変更理由が「好み」「一般論」のものは禁止。briefに根拠がある変更のみ。
- 既存のスタイル/命名/ディレクトリ構造に合わせる。
- 投資助言につながる表現は禁止（「おすすめ銘柄」「買うべき」「必ず儲かる」等）。
- 「予測」ではなく「試算」「前提条件」を用いる。


# PR#28: βローンチ準備（Legal & Feedback Pack）

## 目的
- 外部公開（β）に必要な最低限の法務/サポート/フィードバック導線を整える。
- 「投資助言ではない」ことを明確化し、ユーザーが安心して使える状態にする。

## スコープ
### In scope
- 利用規約ページ（Terms）
- プライバシーポリシーページ（Privacy）
- 免責（投資助言ではない/試算である/最終判断はユーザー）をUIの適切な場所に配置
- 問い合わせ・フィードバック導線
  - 例：Google Form / mailto / GitHub issue など既存方針に合わせる
- アプリ内のフッター/設定等から規約へ遷移できるようにする

### Out of scope
- 決済（Stripe等）実装
- 本格的なCS基盤、メール配信、通知

## 技術固定（変更禁止）
- 文章の法的正確性は“過剰に”追求せず、MVPとして必要十分（ただし投資助言の否定は必須）
- 既存のルーティング/レイアウト規約に合わせる

## 受け入れ条件（AC）
- [ ] /terms と /privacy（または既存規約に沿うURL）で表示できる
- [ ] 主要画面から規約へ到達できる
- [ ] 投資助言ではない旨の免責が表示される（短文で可）
- [ ] フィードバック導線が1クリックで使える

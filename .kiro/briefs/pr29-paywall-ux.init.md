- 既存の仕様・データ・UIを勝手に拡張しない（このPRのIn scopeのみ）。

# PR#29: 課金UX最適化（Paywall Conversion Pack）

## 目的
- Pro価値（再投資率可変、税区分切替、ストレステスト等）を“自然に理解できる”課金UXに整える。
- 403やロック表示が不快にならず、機能差が明確に伝わる状態にする。

## スコープ
### In scope
- ロック表示（Free時）のUX改善
  - 対象：Pro限定機能（再投資率変更、税区分変更、ストレステスト等。現行の権限仕様に合わせる）
  - ロック文言を固定し、「何ができるか」を簡潔に説明
- アップセル導線（最小）
  - 例：プラン説明モーダル/ページ（価格・差分・免責）
  - 価格表示（月額/年額）は定数で良い（決済はしない）
- API 403 のハンドリング統一（エラーJSONに追従、UIが破綻しない）

### Out of scope
- 実際の決済（Stripe/Apple IAP）
- A/Bテスト基盤、計測基盤（PR30以降）

## 技術固定（変更禁止）
- 課金判定はAPI側を正とする（UIだけで判定しない）
- Freeは入力を403で弾くのではなく、丸め・ロック表示を優先（現行仕様を維持）
- 投資助言表現禁止、文言は中立（試算/前提）
- 既存の画面・導線を大きく崩さない（最小の変更）

## UIコピー（固定）
- ロック見出し：Proで利用できます
- 説明（短文）：
  - より現実に近い試算（再投資率・税区分の調整）や、想定外（減配）の影響確認ができます。
- 注意：
  - 入力前提に基づく試算であり、投資助言ではありません。

## 受け入れ条件（AC）
- [ ] FreeでPro機能に触れた際、破綻せずロック表示＋導線が出る
- [ ] Proの価値（何が増えるか）が1画面で理解できる
- [ ] API 403 時の挙動が統一される（変なエラーにならない）
- [ ] 文言が助言・煽りになっていない

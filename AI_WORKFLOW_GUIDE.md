# AI Workflow Guide: Claude x Codex

このリポジトリは、**Claude Codeが設計し、Codex CLIが実装する**という分業体制で構成されています。

## 🚀 使い方

### 1. Claude Code の起動
プロジェクトルートで `claudecode` を起動します。

### 2. 依頼の出し方
あなたは「何を作りたいか」を伝えるだけです。詳細な実装は彼らがやります。

- **新機能の実装**:
  > "src/features/user にユーザープロフィール画面を作りたい。設計から実装までCodexを使ってやって。"
- **リファクタリング**:
  > "src/utils のコードが乱雑なので整理して。Codexを使って型定義もしっかりやって。"
- **バグ修正**:
  > "ログイン時にエラーが出る。原因を調査して修正までやって。"

### 3. モードの違い
Claudeは状況に応じてコマンドを使い分けます。
- **実装モード**: `codex exec --full-auto ...` (ファイルを書き換えます)
- **相談モード**: `codex exec --full-auto --sandbox read-only ...` (安全に分析だけします)

## ⚠️ 注意点
- コードは原則 `src/` 配下に生成されます。
- 実装が中途半端な場合（TODOが残っている等）は、「TODO禁止ルールを守らせて！」とClaudeに伝えてください。
